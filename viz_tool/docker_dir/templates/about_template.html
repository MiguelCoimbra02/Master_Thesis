Cork Oak Gene Network

The Cork Oak Gene Network tool is designed to facilitate the exploration of gene interactions in Quercus suber, a keystone species of Mediterranean forests. Cork oak plays a vital ecological and economic role, yet its populations face increasing threats due to climate change and environmental stressors.

To better understand the genetic mechanisms behind key traits like secondary growth, we processed large-scale transcriptomic datasets and applied co-expression analysis techniques to construct a gene comprehensive Knowlage Network. Using a combination of data mining, statistical correlation methods, and functional annotation, we identified potential regulatory relationships between genes, offering insights into their biological roles.

This tool builds upon publicly available data and integrates with CorkOakDB, providing an interactive platform for visualizing gene interactions. By transforming static datasets into an accessible knowledge network, we aim to support researchers in generating new hypotheses and advancing the study of cork oak genetics.

1. Data Extraction

The process begins by retrieving sequencing data from the NCBI Database using E-utilities. This data includes SRA files containing raw sequencing data for various BioProjects related to Quercus suber, which were obtained through Illumina or BGISeq sequencing technologies.

The retrieved SRA files are then converted into the FASTQ format, a widely-used file type for storing high-throughput sequencing data. This format enables efficient analysis and processing of large sequencing datasets.

2. Data Preprocessing

2.1 Data Quality Assessment and Cleaning

Before analysis, the raw sequencing reads undergo a quality check using FastQC to assess their overall quality. To enhance read accuracy, Trimmomatic is applied to remove low-quality bases and sequencing adapters. After the cleaning process, a second FastQC run is performed to ensure that the preprocessing steps were effective.

2.2 Read Alignment and Quantification

The preprocessed reads are mapped to a reference genome using STAR, a high-performance RNA-seq aligner. This alignment requires GTF and FNA files, which contain gene annotations and the genome sequence, respectively, both extracted from NCBI. After alignment, FeatureCounts is used to quantify gene expression, generating a counts.txt file that lists the number of reads mapped to each gene.

3. Data Normalization

Gene expression data must often be adjusted for sequencing depth and other biases. This normalization is achieved using DESeq, which processes the count data and adjusts expression values accordingly. The resulting normalized data is crucial for downstream network inference.

4. Network Prediction

With the normalized expression data, Seidr is used to infer a gene co-expression network through a multi-method approach that combines several algorithms. This step plays a crucial role in uncovering interactions that may drive important biological processes, providing insights into gene co-expression relationships.

5. Gene Annotation and TF Identification

A gene annotation file (CSV) extracted from NCBI provides additional functional information about genes in the dataset, offering insight into the route of influence of such nodes.

A list of putative transcription factors (TFs) for Quercus suber was acquired in project X [REFERENCE]. Further clarification on this was discussed with Pedro.

6. TF-Gene Interaction Prediction

To refine the network and identify regulatory interactions, ConnecTF is used. This tool integrates transcription factor information to link regulatory elements with their potential gene targets for some species, including Arabidopsis thaliana.

This step of the analysis was performed through homology.

7. Motif Enrichment Analysis

To further validate regulatory interactions, FIMO Motif Analysis searches for cis-regulatory motifs in promoter regions of the putative target genes, providing additional support for predicted TF-gene interactions.

8. Output Generation for Visualization

Finally, the processed network data is formatted into edges.csv and nodes.csv, which serve as inputs for the visualization tool. These files define the network structure, linking genes and transcription factors based on their inferred relationships.
